# Core Beliefs

These beliefs define the target operating model for this repository.

1. The repository, not chat history, is the system of record.
2. A short repo map is better than a long instruction dump.
3. Domain boundaries matter more than framework boundaries.
4. Route handlers should parse and delegate, not own business logic.
5. Large structural work should leave behind a checked-in plan and decision trail.
6. If a rule matters repeatedly, promote it from prose into code or CI.
7. Local validation must be easy enough for agents to run continuously.
